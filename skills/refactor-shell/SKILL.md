# Shell 重构 SOP

> 当用户提出“重构 Shell 脚本”、“整理 Bash 脚本结构”、“提取公共函数”等请求时使用。

## 前置：需求澄清

> 必须先执行 `requirement-clarify-shell`，至少确认 `RFS-01` ~ `RFS-03`。

## 阶段一：确认范围与约束

### 步骤 1：明确重构目标
- 是为了提高清晰度、复用性、兼容性还是安全性
- 涉及哪些脚本、函数、参数解析或命令链
- 哪些对外行为、参数、输出和副作用必须保持不变

### 步骤 2：先输出重构方案

```markdown
## 重构方案
- 重构目标：
- 影响范围：
- 保持不变的行为：
- 分步实施计划：
- 回归验证方式：
```

## 阶段二：小步实施

> 编码时遵循 `rules/13-shell-scripting.mdc`；涉及安全和危险命令时叠加 `rules/14-shell-security.mdc`。

### 步骤 3：按小步推进
- 先提取函数，再整理参数解析，再整理日志与清理逻辑
- 一次只做一类结构调整
- 不顺手修改脚本语义、参数名和输出格式
- 危险命令和路径保护逻辑变化必须单独评估

## 阶段三：回归验证与交付

### 步骤 4：验证重点
- `shellcheck` 和相关测试是否通过
- 参数、输出、退出码和副作用是否保持一致
- 临时资源、权限和清理逻辑是否可靠

### 步骤 5：输出结果

```markdown
## 重构摘要
- 目标：
- 改动脚本：
- 保持不变的行为：
- 验证结果：
- 残余风险：
```

### 步骤 6：任务交付文档
