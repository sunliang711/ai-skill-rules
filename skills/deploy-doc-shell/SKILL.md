# Shell 部署文档 SOP

> 当用户提出“生成 Shell 部署文档”、“整理脚本上线说明”等请求时使用。

## 前置：需求澄清

> 必须先执行 `requirement-clarify-shell`，至少确认 `DS-01` ~ `DS-02`。

## 步骤 1：确认部署范围
- 涉及哪些脚本、定时任务、CI 作业、systemd / cron 配置
- 运行环境、执行用户、工作目录和依赖命令是什么
- 是否包含权限变更、目录初始化、下载步骤或回滚脚本

## 步骤 2：提取关键信息
- 脚本入口、参数、环境变量和依赖命令
- 必需目录、文件权限、日志路径、临时目录
- 发布顺序、验证步骤和回滚方式

## 步骤 3：生成文档

```markdown
## Shell 部署说明
- 脚本 / 任务清单：
- 执行环境：
- 参数与环境变量：
- 权限与目录要求：
- 发布顺序：
- 验证步骤：
- 回滚方案：
```

## 步骤 4：交叉验证
- 路径、命令、用户和权限要求必须与实际脚本一致
- 标明哪些步骤是高风险或需要人工确认

## 步骤 5：任务交付文档
在 `docs/deploy/` 下生成：`docs/deploy/{YYYY-MM-DD}-deploy-shell-{脚本简称}.md`
