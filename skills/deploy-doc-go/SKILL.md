# Go 部署文档生成 SOP

> 当用户提出“生成 Go 服务部署文档”、“整理这次发布需要的配置和步骤”、“列出新增环境变量和迁移项”等请求时使用。

## 前置：需求澄清

> 必须先执行 `requirement-clarify-go`，至少确认 `DG-01` ~ `DG-02`。

## 步骤 1：确定变更范围
- 用户指定文件或模块。
- Git diff。
- 本次对话改动过的 Go 服务相关文件。

重点扫描：
- `go.mod` / `go.sum`
- `config/`、`config.toml`、`private.toml`
- `cmd/`、`internal/`
- `Dockerfile`、K8s YAML、systemd 脚本
- SQL / migration 文件
- `.env.example`、部署脚本、CI/CD 配置

## 步骤 2：逐项提取部署信息

### 2.1 环境变量
记录：
- 变量名
- 用途
- 示例值
- 是否敏感
- 影响服务

### 2.2 配置项
重点关注：
- `mapstructure` 对应配置
- 默认值与是否按环境调整
- 启动必填项和可选项

### 2.3 依赖与中间件
记录：
- `go.mod` 新增或升级依赖
- DB、Redis、MQ、对象存储、外部 API
- 新端口、健康检查、路由前缀、定时任务

### 2.4 数据与迁移
记录：
- migration 执行顺序
- 是否需要数据回填
- 是否可回滚

### 2.5 运行与发布方式
记录：
- Docker / K8s / systemd / 二进制直跑
- 启动命令
- 构建产物
- 发布顺序与依赖前置条件

## 步骤 3：生成部署文档

```markdown
# 部署文档

## 基本信息
- 功能/版本：
- 发布日期：
- 涉及服务：
- 部署环境：

## 一、环境变量变更
| 变量名 | 用途 | 示例值 | 是否敏感 | 影响服务 |
|--------|------|--------|:---:|----------|

## 二、配置项变更
| 配置路径 | 默认值/当前值 | 说明 | 是否需按环境调整 |
|----------|---------------|------|:---:|

## 三、依赖与中间件变更
| 类型 | 名称 | 变更说明 |
|------|------|----------|

## 四、数据库/迁移变更
| 变更类型 | 对象 | 说明 | 是否可回滚 |
|:---:|------|------|:---:|

## 五、部署步骤
1. [步骤]
2. [步骤]

## 六、验证清单
- [ ] 服务启动正常
- [ ] 健康检查通过
- [ ] 核心接口冒烟通过

## 七、回滚方案
1. [步骤]
2. [步骤]

## 八、注意事项
- [风险或说明]
```

## 步骤 4：交叉验证
- 文档里的环境变量是否覆盖代码所需项。
- 配置路径、默认值、敏感标记是否准确。
- 迁移顺序与回滚方案是否可执行。
- 部署步骤是否符合真实启动方式。

## 步骤 5：任务交付文档
在 `docs/` 下生成：`docs/{YYYY-MM-DD}-deploy-go-{功能或版本简称}.md`

建议包含：
- 变更范围
- 提取来源
- 部署文档正文
- 交叉验证结果
- 风险与后续建议
