# Shell Bug 修复 SOP

> 当用户提出“修复这个 Shell 脚本”、“排查脚本报错”、“定位 CI 脚本异常”等请求时使用。

## 前置：需求澄清

> 必须先执行 `requirement-clarify-shell`，至少确认 `BS-01` ~ `BS-03`。

## 阶段一：问题理解与定位

### 步骤 1：整理已知信息
- 报错、退出码、执行日志、运行环境
- 输入参数、环境变量、当前目录、依赖命令
- 期望行为和影响范围

### 步骤 2：按 Shell 常见根因定位
- 引号缺失、空变量、路径带空格
- `set -euo pipefail` 下的错误传播问题
- 命令不存在、权限不足、环境差异
- 临时文件未清理、`trap` 缺失、危险删除保护不足
- bash / sh / zsh 兼容性差异

### 步骤 3：先输出定位结论

```markdown
## Bug 定位分析
- 问题现象：
- 根因位置：
- 触发条件：
- 修复思路：
- 影响评估：
```

## 阶段二：修复实施

> 修复时必须遵循 `rules/13-shell-scripting.mdc`；涉及路径、权限、下载、删除和命令执行安全时，同时遵循 `rules/14-shell-security.mdc`。

### 步骤 4：最小化修复
- 只改与问题直接相关的部分
- 保持参数、输出和副作用尽量稳定
- 涉及危险命令时优先增加保护，不扩大脚本行为

## 阶段三：验证与交付

### 步骤 5：验证修复有效性
- 优先复现问题再验证修复
- 运行 `shellcheck` 或项目既有检查
- 验证正常路径、错误路径和保护分支

### 步骤 6：输出结果

```markdown
## Bug 修复摘要
- 问题：
- 根因：
- 修复方式：
- 影响范围：
- 验证方式：
- 回归风险：
```

### 步骤 7：任务交付文档
在 `docs/delivery/` 下生成：`docs/delivery/{YYYY-MM-DD}-bugfix-shell-{问题简称}.md`
