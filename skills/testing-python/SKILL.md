# Python 测试生成 SOP

> 当用户提出“给这个 Python 模块写测试”、“补 pytest 用例”等请求时使用。

## 前置：需求澄清

> 必须先执行 `requirement-clarify-python`，至少确认 `TP-01` ~ `TP-02`。

## 步骤 1：分析被测目标
- 明确被测模块、函数、路由、任务或脚本的核心职责
- 找出正常路径、错误路径、边界条件和权限/超时场景
- 判断依赖该 mock、fake 还是走隔离环境

## 步骤 2：先输出测试设计

```markdown
## 测试用例设计
- 目标文件 / 模块：
- 测试类型：单元 / 接口 / 集成 / 任务 / 脚本
- 关键场景：
- 依赖隔离策略：
- 运行命令：
```

## 步骤 3：编写测试
- 优先使用 `pytest`
- Web 层测试优先使用测试客户端
- 外部依赖优先使用 fixture、mock、fake 或隔离环境
- 覆盖正常路径、错误路径、边界条件和权限/超时场景
- 避免让单元测试依赖真实外部服务

## 步骤 4：输出交付结果

```markdown
## 测试完成摘要
- 测试文件：
- 覆盖目标：
- Mock / Fake 依赖：
- 运行命令：
- 未覆盖场景：
```

### 步骤 5：任务交付文档
在 `docs/` 下生成：`docs/{YYYY-MM-DD}-test-python-{模块简称}.md`
