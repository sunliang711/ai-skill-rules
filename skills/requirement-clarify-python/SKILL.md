# Python 需求澄清协议

> 本协议被 `feature-dev-python`、`bug-fix-python`、`refactor-python`、`code-review-python`、`testing-python`、`deploy-doc-python` 引用。

## 核心原则

- 必问项未确认前，禁止直接输出方案、代码或结论
- 每轮追问控制在 3~5 个关键问题
- 允许合理推断，但必须明确标注假设
- 所有 🔴 必问项确认后，才能进入正式执行阶段

## 执行方式

1. 先在内部判断每项为“已知 / 未知 / 可推断”
2. 将未知的必问项和关键选问项组成一轮追问
3. 最多 3 轮；剩余少量模糊点可带假设推进
4. 退出前先复述 Python 任务理解

## 专属问题清单

### feature-dev-python

#### 🔴 必问项
- FP-01：业务目标与边界是什么
- FP-02：涉及哪些包、模块、服务或脚本
- FP-03：入口形态是什么：HTTP、任务、CLI、库能力还是数据处理
- FP-04：关键规则、校验、状态流转和依赖要求是什么

#### 🟡 选问项
- FP-05：是否涉及数据库、缓存、队列、对象存储或外部 API
- FP-06：是否有性能、时延、吞吐或内存约束
- FP-07：是否有现成实现或文档可参考

### bug-fix-python

#### 🔴 必问项
- BP-01：问题现象是什么，报错或异常行为是什么
- BP-02：复现条件是什么
- BP-03：期望行为是什么

#### 🟡 选问项
- BP-04：影响范围与紧急程度
- BP-05：最近相关变更与日志线索

### code-review-python

#### 🔴 必问项
- RP-01：审查范围是哪些文件、模块或 PR
- RP-02：业务背景和改动目的是什么

#### 🟡 选问项
- RP-03：优先关注安全、性能、规范还是测试
- RP-04：是否有已知风险点

### refactor-python

#### 🔴 必问项
- RFP-01：重构目标是什么
- RFP-02：涉及哪些模块、类、函数或接口
- RFP-03：是否要求外部行为完全不变

#### 🟡 选问项
- RFP-04：回归保护和现有测试情况
- RFP-05：哪些配置、协议、数据结构不允许动

### deploy-doc-python

#### 🔴 必问项
- DP-01：涉及哪些服务、脚本、worker、定时任务
- DP-02：目标环境和部署方式是什么

#### 🟡 选问项
- DP-03：是否包含迁移、消息消费、缓存预热
- DP-04：回滚与停机要求

### testing-python

#### 🔴 必问项
- TP-01：要测试哪些模块、类、函数或路由
- TP-02：测试类型是单元、集成、接口还是任务/脚本

#### 🟡 选问项
- TP-03：必须覆盖的边界、异常和权限场景
- TP-04：外部依赖该如何隔离
