# Python 部署文档 SOP

> 当用户提出“生成 Python 部署文档”、“整理 Python 发布清单”等请求时使用。

## 前置：需求澄清

> 必须先执行 `requirement-clarify-python`，至少确认 `DP-01` ~ `DP-02`。

## 步骤 1：确认部署范围
- 涉及哪些服务、worker、定时任务、CLI 或脚本
- 目标环境、运行方式和依赖服务是什么
- 是否包含迁移、消息消费、缓存预热、计划任务或对象存储配置

## 步骤 2：提取关键信息
- 启动命令、构建步骤和运行时依赖
- 环境变量、配置文件、密钥来源
- 端口、队列、数据库、缓存和权限要求
- 发布顺序、健康检查、回滚步骤

## 步骤 3：生成文档

```markdown
## Python 部署说明
- 构建 / 安装步骤：
- 服务与脚本清单：
- 配置与环境变量：
- 数据迁移 / 初始化：
- 发布顺序：
- 验证步骤：
- 回滚方案：
```

## 步骤 4：交叉验证
- 命令、路径、服务名和变量名必须与代码一致
- 标明哪些步骤依赖人工确认或外部平台

## 步骤 5：任务交付文档
在 `docs/` 下生成：`docs/{YYYY-MM-DD}-deploy-python-{服务简称}.md`
