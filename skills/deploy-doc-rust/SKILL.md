# Rust 部署文档 SOP

> 当用户提出“生成 Rust 部署文档”、“整理 Rust 发布清单”等请求时使用。

## 前置：需求澄清

> 必须先执行 `requirement-clarify-rust`，至少确认 `DR-01` ~ `DR-02`。

## 步骤 1：确认部署范围
- 涉及哪些 crate、二进制、服务、脚本和配置
- 部署到什么环境、以什么方式运行
- 是否包含数据库迁移、消息订阅、任务调度或 systemd / 容器更新

## 步骤 2：提取关键信息
- 构建命令与产物路径
- 环境变量、配置文件、密钥来源
- 依赖服务、端口、网络、存储和权限
- 启停方式、健康检查、回滚步骤

## 步骤 3：生成文档

```markdown
## Rust 部署说明
- 构建步骤：
- 产物清单：
- 配置与环境变量：
- 数据迁移 / 初始化：
- 发布顺序：
- 验证步骤：
- 回滚方案：
```

## 步骤 4：交叉验证
- 命令、路径、服务名、环境变量与实际代码保持一致
- 明确哪些步骤依赖人工确认

## 步骤 5：任务交付文档
在 `docs/` 下生成：`docs/{YYYY-MM-DD}-deploy-rust-{服务简称}.md`
