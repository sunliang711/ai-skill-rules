# Shell 测试生成 SOP

> 当用户提出“给这个 Shell 脚本补测试”、“写 bats 用例”、“补验证步骤”等请求时使用。

## 前置：需求澄清

> 必须先执行 `requirement-clarify-shell`，至少确认 `TS-01` ~ `TS-02`。

## 步骤 1：分析被测目标
- 明确脚本、函数、参数分支和副作用
- 找出正常路径、错误路径、空输入、非法输入和危险操作保护场景
- 判断哪些外部命令需要 mock，哪些路径需要临时目录隔离

## 步骤 2：先输出测试设计

```markdown
## 测试用例设计
- 目标脚本：
- 测试类型：shellcheck / bats / 手工验证 / 组合
- 关键场景：
- Mock / 隔离策略：
- 运行命令：
```

## 步骤 3：编写测试或验证脚本
- 静态检查优先运行 `shellcheck`
- 复杂流程优先补 `bats`
- 外部命令和文件系统副作用优先隔离到临时目录
- 覆盖正常路径、错误路径和危险操作保护分支

## 步骤 4：输出交付结果

```markdown
## 测试完成摘要
- 测试文件 / 验证脚本：
- 覆盖目标：
- Mock / 临时目录策略：
- 运行命令：
- 未覆盖场景：
```

### 步骤 5：任务交付文档
在 `docs/delivery/` 下生成：`docs/delivery/{YYYY-MM-DD}-test-shell-{脚本简称}.md`
